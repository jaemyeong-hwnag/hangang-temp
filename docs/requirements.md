## 2. 기능 요구사항 (Functional Requirements)

### 2.1 위험도 판정 시스템 [FR-001]

**우선순위:** Critical

| 단계 | 조건 | 메시지 | 색상 |
|------|------|--------|------|
| 안전 | 수온 ≥ 15°C AND 코스피 변동률 ≥ 0% | 오늘은 한강 패스 | green |
| 보통 | 수온 10~15°C OR 변동률 -1~0% | 그럭저럭 버틸만함 | yellow |
| 주의 | 수온 5~10°C OR 변동률 -3~-1% | 라면이나 한 개 끓여두세요 | orange |
| 위험 | 수온 < 5°C AND 변동률 < -3% | 이미 늦었을 수도 | red |

- 위험 판정 시 인터스티셜 광고 노출 트리거

### 2.2 라면 지수 계산 [FR-002]

**우선순위:** High

- 수온 점수 (0~50): `max(0, (20 - 수온) * 2.5)`
- 코스피 점수 (0~50): `max(0, min(50, 변동률 * -10))`
- 라면 지수 = 수온 점수 + 코스피 점수 (0~100)
- 라면 필요량으로 표현: 0~25 → 0개, 26~50 → 1개, 51~75 → 2개, 76~100 → 3개

### 2.3 7일 히스토리 차트 [FR-003]

**우선순위:** High

- 수온과 코스피 이중 축 라인 차트
- Chart.js 사용
- 데이터: 로컬스토리지에 누적 저장 (서버 없음)

### 2.4 푸시 알림 [FR-004]

**우선순위:** Medium

- 트리거 조건:
  - 코스피 -3% 이하 실시간 감지
  - 수온 10°C 이하
  - 오전 9시 모닝 브리핑 (Service Worker Scheduled)
- Web Push API 사용 (VAPID 키 필요 — Vercel 환경변수로 관리)

### 2.5 소셜 공유 [FR-005]

**우선순위:** High (바이럴 핵심)

- 공유 버튼: 판정 결과 배너 직하단
- 공유 내용: 위험도 단계 + 수온 + 코스피 변동률 텍스트
- `navigator.share()` 사용 (Web Share API)
- 폴백: 클립보드 복사
- OG 메타태그: 정적 설정 (동적 렌더링 v2.0에서 추가)

### 2.6 광고 [FR-006]

**우선순위:** Medium

| 위치 | 형식 | 트리거 |
|------|------|--------|
| 판정 결과 직후 (위치 A) | 반응형 디스플레이 | 페이지 로드 |
| 위험 단계 진입 시 (위치 B) | 인터스티셜 | 위험 판정 |
| 알림 클릭 후 랜딩 (위치 C) | 앵커/반응형 배너 | 알림 딥링크 |

## 3. 비기능 요구사항 (Non-functional Requirements)

### 3.1 성능 [NFR-001]
- 초기 로딩: Lighthouse Performance ≥ 85
- LCP < 2.5s on 4G
- 번들 사이즈: gzip 기준 < 200KB (광고 스크립트 제외)

### 3.2 접근성 [NFR-002]
- 색맹 대응: 색상 외 아이콘/텍스트로 위험도 구분
- 모바일 우선 반응형 (375px ~ 430px 기준)

### 3.3 오프라인 [NFR-003]
- PWA: 마지막 조회 데이터 캐시 (Service Worker)
- 오프라인 시 "데이터 없음" 상태 명시

### 3.4 개인정보 [NFR-004]
- 서버 로그 없음
- 사용자 데이터 로컬스토리지에만 저장
- AdSense 쿠키: GDPR/국내법 준수 배너 추가 (v1.1)

## 4. 데이터 요구사항

### 4.1 한강 수온
- 출처: 한국수자원공사 물정보 포털 (water.nier.go.kr)
- 갱신 주기: 1시간
- 측정 지점: 한강대교 (대표 지점)
- API: 공공데이터포털 `금강/영산강/한강/낙동강 수질 정보 API`

### 4.2 코스피 지수
- 출처: Yahoo Finance (`^KS11`)
- 방식: Vercel Edge Function (`/api/kospi`) 프록시 → CORS 우회
- 갱신 주기: 15분 (장 중), 1일 (장 마감 후)

---

## 5. 제약사항

- 서버 인프라 비용 없음 (Vercel 무료 티어)
- CORS 허용 엔드포인트 우선 사용
- Edge Function은 KOSPI 프록시 용도로만 최소 사용
